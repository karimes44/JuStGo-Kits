JuStGo Kits

What are JuStGo Kits?

Each JuStGo kit contains all you need to build a game or an application from scratch.

It is helpful if you know a little bit about HTML, CSS, and JavaScript, but don't worry, the kits contain links to helpful resources to catch up on what you need to build the game or even to learn as you go!

The kit also contains tips and explanations that help you complete the task. At the end, a solution proposal is given for this task. You can decide for yourself if you want to continue with your own solution, switch to the provided one, or to mix-and-match.

The provided proposal solution solves the task with very little code so that you can clearly see what is done and how. As a consequence, it may look a bit plain. Let this inspire you to spice it up!
Go wild with CSS and add HTML elements as you like. Make it yours!

Just like with a Lego kit, you can take the building blocks from the JuStGo kits and reassemble them your way, either to make a game from a kit better or even to create your own game. At the end of each kit you will find challenges and ideas for that.

You can work with JuStGo kits on any browser and any OS, but you will need a code/text editor. Check out some recommendations here.

Did you attend the Code Nation Intro to Web Development class?

If so, you can think of the first two JuStGo kits as a refresher, as they cover a similar set of content.

... but there are difference:

  • The JuStGo kits only use JavaScript instead of jQuery. Don't worry, all you can do with jQuery, you can do with JavaScript, and you'll gain a deeper understanding on what is going on.
  • JuStGo kits are written to run locally and not on a web service, like PopCode, but you can create your projects inside PopCode, if you like.

You see, even if you completed the Intro to Web Development class, you will find new things in the first two kits.

List of JuStGo Kits

  • Home Page: Build a Home Page for yourself or for someone you admire - and learn about HTML and CSS.

  • Choose your own Adventure: Build an interactive story book - and learn about Javascript events and handler.

  • 15 Squares: A sliding puzzle having 15 square tiles numbered 1-15 in a frame that is 4 tiles high and 4 tiles wide, leaving one unoccupied tile position. Tiles in the same row or column of the open position can be moved by sliding them horizontally or vertically, respectively. The goal of the puzzle is to place the tiles in numerical order.

  • Mastermind: A single player a code-breaking game. You have 10 tries to break the code. After each guess you get additional clues to help you refine your next guess.

  • Pong: A table tennis-themed arcade-type video game originally released in 1972. It was one of the earliest arcade video games.

  • Warehouse: You guide a warehouse worker, who needs to push heavy pallets onto the certain spots.

Technical Requirements

The games and apps you will write will run on any browser and you can use any text editor to write the games and apps.

You can use a PC, a Mac, or a Chrome Book (... even a Linux box) to write the games and apps.

On mobile devices like Smart Phones and Tablets you can run the games, but you can't really write them there. That's because it is hard to find good text editors for mobile devices and the mobile browser don't have Developer Tools.

Recommended Browser

Google Chrome works pretty well as the browser, and it support a good code editor as an free app (see below).

Developer Tools

When you write code, it is very normal that it does not quite work the first time (...like never!), and frankly, figuring out what is wrong can be confusing (sigh) - but also a fun challenge!
So, don't get discouraged, there are simple things you can do to get your code to start working right.

For example, you can check the value of some variables that don't quite do what you expected in a pop-up alert alert(rogueVariable) or with a console.log(rogueVariable) statements in the JavaScript code. You can un-comment (=make them active again) them as needed or add new once to show the variables you are confused about.

There are also the Developer Tools - most browsers have them built-in. In Chrome you can activate the Developer Tools with Ctrl-Shift-i, or from the customization tab > More Tools > Developer tools.

The Console panel in the Developer Tool window you can see a list of all coding mistakes that prevent the code from running at all.

Another Tip: Browsers are designed to "muddle through" if there is an error on some Web page. When your code behaves like "totally weird", it may just be that there is a mistake in your code somewhere else and the browser just tried to limp along. Check in the Developer Tools if you can find any hints of what may have tripped up the browser.

In the Source panel you can step through the running code and see all the variables with their current values and see in which order your code statements are executed. Then you learn pretty quick why things are not quite working as you wanted. And once you know what is going wrong - fixing it is often the easy part!

If you need a quick intro to the Chrome Developer Tools, search for a video on YouTube, like for example this one: Debugging JavaScript - Chrome DevTools 101.

JuStGo Kit: Choose you own Adventure

Introduction

Working with the JuStGo Kit

In this JuStGo kit you will build a Choose your own Adventure story. The kit will lead you step-by-step through each task that needs to be completed to build the entire interactive story.

The JuStGo kit introduces you to several different JavaScript event handler in the context of a working example. You can then use them in your own projects, if you wish.

Each section is formulated as a task to solve. You can find explanations, tips, and resources that help you complete each task. At the end of each section a solution example is given. You can decide for yourself if you want to take over some of the provided example code or stay with your solution.

Note: To see the HTML code, open the solution proposal in your text editor, such as Chrome Caret, or right-click anywhere on the solution proposal page and select View Page Source.

The provided solution example solves the task with very little code so that you can clearly see what is done and how. As a consequence, it may look a bit plain. Let this inspire you to spice it up!

Go wild - make it yours!

Task 0: Create the HTML placeholder with placeholder frames

Create an HTML placeholder page that...

  • defines the page document type to be HTML5,
  • has a header section with a title tag,
  • has a style section, and
  • has a body section.

When you step through your story, you will make the current page or frame of your story visible, while you will hide all others. Therefore, add to template frames to the HTML page - you can use <div> tags for that.

Objectives

This task uses only content that was already introduced in the Home Page JuStGo kit.

Tips and Resources

You will need to selectively make one <div> frame visible while keep all other hidden. You can do that by assign the first frame to a "shown" class, and all others one to a "hidden" class. You will also need to give each frame it's own id attribute.

In CSS section, style the <div> the way you like them, and define the class .hidden

Extra tip: It is easier to work with your story board if all frames are visible while you work on the code. Instead of actually hiding them you could just give the hidden frames a special background color, so you know that these frames are meant to be hidden in the current stage of your story.

Once you are comfortable with you code, you just add display: none; to the .hidden CSS definition.

Task 2: Trigger an action via onclick

Create buttons to make a choice between advancing to the next frame or not playing the game.

Objectives
  • Working with buttons
  • Working with the onclick event.
  • Defining functions for event handling.
Tips and Resources

You can find an introduction to Events at W3 School: JavaScript HTML DOM Events, and a demo of the onclick event and its event handler in this example.

You can use similar code to define your buttons, events, and event handler.

At least one of your event hander should re-assign the first frame to the hidden the class, and the second to the shown class.

You can use JavaScript - HTML DOM Methods to connect the JavaScript with the HTML. In particular, the method .innerHTML lets you read and write text from JavaScript to an HTML element. The method .setAttribute allows you to re-assign an html element to a different class.

Solution Proposal

Highlights of added code

Two buttons are introduced. Each button is given the onclick attribute with a value that is the name of a function.

Then you need to define these functions.

The enterDream()" function just moves you along to the next frame by swapping the classes. (Remember: For easier debugging, at this stage just the background color is changed.)

The mindBusiness()" function replaced the content of the entire frame with a Good bye message.

<!-- _______________________________________________________________ -->
<div id="Frame1" class="shown">
<p>Alice was sleeping in the wood when she she noticed something weird...</p>

<button onclick="enterDream()">Click here to enter in her dream</button>
<button onclick="mindBusiness()">Click here to mind your own business</button>
<br><br>
<img src="./images/sleeping.jpg" alt="Alice sleeping" class="image1">
</div>

<script>
function enterDream(){
 document.getElementById("Frame1").setAttribute("class","hidden");
 document.getElementById("Frame2").setAttribute("class","shown");
}
function mindBusiness(){
 document.getElementById("Frame1").innerHTML = "<p>(sign) ... 
    to bad you are such a busy bee... please do come back when you have more time!</p>
    <h2>The End</h2>";
}
</script>

The JavaScript function document.getElementById("Frame1") lets you access an HTML element (also called DOM) by its id - here "Frame1". Remember that an id has to be unique, so you can be sure that this function will return only one element.

Then you can use the methods .setAttribute("class","hidden") to change an attribute of the HTML element here the class attribute is re-set to "hidden".

With the .innerHTML method you can read the content of an HTML element if use it on the left side of an variable assignment statement, or you can replace the content of that HTML element if you use it on the right side of an assignment statement, which is what is done here.

Note: The JavaScript used in this project <script> could be placed anywhere in the code, e.g., all the JavaScript could be placed in the beginning into a single section - and that is how it is usually done to keep the code all together and make it even possible to place it in a separate file. When using this approach you have to take care that the HTML is fully loaded first before the part of the JavaScript code is executed (in later JuStGo kit you will come across that).

For this project it may be more convenient to place the JavaScript code for each frame right next to the frame.

Solution Proposal

Highlights of added code

To define a drop-down menu you embed in the <select> tag the different option (with the <option> tag) that you want to provide. The content of the <option> tag is what is displayed on the page, but the value of the value atrribute is what you can retrieve from your JavaScript code. 

<div id="Frame3" class="hidden">
<p><q>Where did it go?</q> wondered Alice. <q>It must be in that hole...</q></p>
<select id="YoN" onchange="readYoN()">
 <option value="answerIsDC">Select Answer</option>
 <option value="answerIsDC">Don't no ...</option>
 <option value="answerIsYes">Yes</option>
 <option value="answerIsNo">No</option>
</select>
<br><br>
<img src="./images/Hole.jpg" alt="Alice and the rabbit hole" class="image1">
</div>

You can retrieve what you have selected in the pull-down menu with the .value method. Note that this will give you the value of the value attribute of the <option>and not the content of it!

Then you can use a conditional block to decide how to handle the event.

<script>
function readYoN(){
 let answer = document.getElementById("YoN").value;
 if (answer === "answerIsYes") {
  document.getElementById("Frame3").setAttribute("class","hidden");
  document.getElementById("Frame4").setAttribute("class","shown");
 } else if (answer === "answerIsNo") {
  document.getElementById("Frame3").setAttribute("class","hidden");
  document.getElementById("Frame1").setAttribute("class","shown");
 } else if (answer === "answerIsDC") {
  alert("Make up your mind, please...");
 }
}
</script>

Note: The event is only triggered with you actually change the option in the pull-down. So if you come back to the frame without reloading the entire page, then your last selection may still be shown. Then you have to select something else first before you can re-select what you had before - a bit of a nuisance...

Tips and Resources

This link gets you to an overview of CSS Animations, and this one to an example that showcases the approaches you will need for this task.

Solution Proposal

Highlights of added code

For the animation create a new CSS class, say .rotate and define the animation-name, how long a single animation should last (animation-duration), and how often it should repeat (animation-iteration-count):

<style>
...
.rotate {
 animation-name: rotating;
 animation-duration: 2s;
 animation-iteration-count: 3;
}
...

Then use the @keyframes command to define the animation:

<style>
...
@keyframes rotating {
 10% {transform: rotate( 36deg); width: 360px;}
 20% {transform: rotate( 72deg); width: 320px;}
 30% {transform: rotate(108deg); width: 280px;}
 40% {transform: rotate(144deg); width: 240px;}
 50% {transform: rotate(180deg); width: 200px;}
 60% {transform: rotate(216deg); width: 160px;}
 70% {transform: rotate(252deg); width: 120px;}
 80% {transform: rotate(288deg); width:  80px;}
 90% {transform: rotate(324deg); width:  40px;}
100% {transform: rotate(360deg); width:  20px;}
}
</style>

The list of percentages define how far into the animation the given CSS properties will be changes. So 10% means that after 10% of the 2s duration, that is 0.2s the first set of CSS property changes will be applied.

So here after each additional 10% you rotate the picture by an additional 36° angle, so that when the first animation look completes, it the picture turned around once. At the same time you keep reducing the size.

Now you have to apply this animation to an HTML element, here an image. There is a bit of a problem with when the animation is triggered. You don't want to start it when the page is loading, because at that time the image is hidden.

You can just assign the rotate class to the image just when you make it shown:

  document.getElementById("Frame4").setAttribute("class","shown");
  document.getElementById("falling").setAttribute("class","rotate");
 } else if (answer === "answerIsNo") {
  ...
 }
}
</script>

Task 8: onmouseover and onmouseout

The Cheshire Cat is a strange creature, when you address her, she vanishes, and when turn away she comes back.

Can you program that?

Objectives
  • The onmouseover event
  • The onmouseout event
  • The <span> tag
  • Using this
Tips and Resources

The onmouseover event lets you trigger an event once the mouse pointer entered the HTML object, and the onmouseout event is trggerend the the pointer leaves the HTML object again.

Here is an example of using the onmouseover and onmouseout events.

Also check out this: The JavaScript this Keyword

.

Solution Proposal

Highlights of added code

For the first image just the good old onclick event is use, nothing new here...

<div id="Frame8" class="hidden">
...
<img src="./images/Catapillar.jpg" alt="Catapillar" class="image2" onclick="goToParty()">
...

But a lot is going on with the second one:

...
<span id="vanishingCat">
<img src="./images/CheshireCat1.jpg" alt="CheshireCat" class="image2"
onmouseover="dim(this)" onmouseout="comeback(this)" onclick="vanish()"></span>
</div>

First a <span> tag is wrapped around the image - you'll see below why that is needed. A <span> is another neutral tag just like <div>, but you can use it within in a line.

Wow, now we have three event handles, our old friend onclick, and our two new friends onmouseover and onmouseout.

But what is this?

Just like the word this lets you refer to an object without calling it by name, the JavaScript keyword this allows an object to refer to itself. Here the this argument inside the function call dim(this) acts as the address of <img> tag, so that you can use to change the src attribute of it (here you swap out the image):

<script>
function dim(obj) {
 obj.setAttribute("src", "./images/CheshireCat2.jpg");
}
function comeback(obj) {
 obj.setAttribute("src", "./images/CheshireCat1.jpg");
}

function vanish() {
 document.getElementById("vanishingCat").innerHTML= 
    "<q>You can't get anything from that cat - how rude...</q> thought Alice."
} 

function goToParty() {
 alert('"Well, I would go the never ending tea party" said the catapillar')
 document.getElementById("Frame8").setAttribute("class","hidden");
 document.getElementById("Frame9").setAttribute("class","shown");
}
</script>

You could have used an id tag as earlier, but this is way cooler, right?

When you finally click on the picture the vanishes completely, so you need to remove the entire <img> tag. An easy way of doing that is to replace the HTML content of the parent tag. The <span> tag serves as this parent tag.

Task 9: Trigger an action by pressing a key

Now you made it to he end of this story. Can you generate a list of useful links if a certain key is pressed, and restart the game on another key?

Objectives
  • HTML DOM Element Event Listener
  • The onkeydown Event
  • the KeyboardEvent key Property
Tips and Resources

A keyboard event may not be associated with any particular tag. It just belongs to the entire document, and indeed you can add an event listener to the document.

See:

  • HTML DOM Element addEventListener()
  • onkeydown Event
  • KeyboardEvent key Property

Solution Proposal

Highlights of added code

Previously you have added an event to a tag by giving the tag an event attribute, such as onclick, but you can also an an event listener from JavaScript withe the addEventListener method. Here, the event listener is added to the entire document:

<script>
document.addEventListener('keydown', function (keyPressed) {
...
})
</script>

The event listener checks of a key has been pressed, and if so executes a function. Here, instead of calling a named function, the function is defined right there

document.addEventListener('keydown', function (keyPressed) {
 let k = keyPressed.key;
 // alert('Pressed Key>'+k+'<');
 if (k === "w") {
  html  = '<h2>Thanks for playing</h2>'
  html += '<p>To learn more about Alice in Wonderland, "
  html += 'check out her <a href='
  html += '"https://en.wikipedia.org/wiki/Alice%27s_Adventures_in_Wonderland">'
  html += 'Wikipedia Page</a></p>'
  html += '<p>... and here is a picture of the real Alice:</p>'
  html += '<img src= "https://upload.wikimedia.org/wikipedia/commons/thumb/b/b1/Dressed_in_Her_Best_Outfit.jpg/330px-Dressed_in_Her_Best_Outfit.jpg">'
  document.getElementById("Frame9").innerHTML= html;
 }
 if (k === "r") {
  html  = '<h2>Thanks for playing</h2>\n'
  html += 'To start over click <a href= "ChooseAdventure.html">here</a>.'
  document.getElementById("Frame9").innerHTML= html;
 }
})

Here keyPressed is actually the event handle, and the method .key returns the key that was pressed.

If the key was "w" then a long string is build-up which contains links to Wikipedia and other formatted text. The string, which now contains HTML is then used to replace the entire content of the Frame9 <div> element.

If the key was "r" a similar things happen, just with different text... any other key is ignored.

In order to activate this event listener only when Frame9 is actice, it is a good idea to place the code into the goToParty() function definition.
